What does it mean to display inline vs inline blocks?

To understand the difference between  display: inline  and  display: inline-block  we must first understand the terms "block" and "inline".
(block, inline, and inline-block are all values of the "display" attribute)

block elements are always displayed on a new line from the previous element and will create a line-break afterwards. They can also be given a set height and width, and by default will take up the full available width of the parent element. Some common default block elements are <p>, <div>, and <h1> to <h6>. inline elements are displayed on the same line as the element before them and will only take up as much space as the content they contain. They cannot be given a set height or width (see example below; the height and width styles do nothing). Some common default inline elements are <span>, <a>, and <code>. inline-block elements combine aspects of block and inline together. They are like inline elements in that they will be displayed on the same line as other inline elements and won't create a line-break afterwards, but they are like block elements in that they can be given a set height and width. If height and width are not specified, they will only be as big as the content they contain (like an inline element).
